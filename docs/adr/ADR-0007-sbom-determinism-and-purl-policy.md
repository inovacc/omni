# ADR-0007: SBOM determinism, round-trip oracle & purl-correctness policy

**Status:** Accepted
**Date:** 2026-06-03
**Decision:** `omni sbom` emits byte-deterministic SPDX-2.3 / CycloneDX-1.5 JSON from a pure-stdlib `encoding/json` emitter over hand-written minimal structs. The upstream SPDX/CycloneDX libraries are **excluded from the default build** and used only as an in-process round-trip validator behind `//go:build omni_sbomvalidate`. Go purls follow a fixed normalization policy, and `pkg/sbom/format.Document` is the only cross-package boundary type.

## Context

Phase 05 ships `omni sbom`, producing SPDX-2.3 JSON and CycloneDX-1.5 JSON for (a) a Go module directory (`go.mod` + `go.sum`) and (b) a built Go binary (via `debug/buildinfo`). The phase has two hard requirements that conflict with the obvious "just use the upstream encoder" approach:

1. **Byte-determinism.** Identical input bytes must produce identical output bytes so SBOMs are golden-master pinnable, reproducible in CI, and diffable across builds. This is the foundational value omni promises: deterministic, testable output on every OS with no external processes.
2. **No transitive bloat / no exec.** omni's core constraint is a single static pure-Go binary with no CGO and no spawned processes. Pulling heavy dependency trees into the default build path violates the supply-chain posture omni is meant to *improve*.

The upstream encoders (`github.com/spdx/tools-golang`, `github.com/CycloneDX/cyclonedx-go`) fail both: they inject random `serialNumber`s and wall-clock timestamps, do not guarantee stable field/key ordering, and pull large transitive trees. The Phase 04 MVS finding confirmed that even build-tag-gated heavy dependencies bump `go.mod` via minimal version selection — so they cannot simply be "hidden behind a tag" and forgotten; they must be quarantined to a build that omni releases never compile.

This ADR is the **hard gate** for Phase 05: it locks the decisions below before any emitter, collector, or CLI code lands. The spec (`docs/superpowers/specs/2026-05-16-05-pkg-sbom-design.md`) declares "Required before any code lands (1 ADR)" — non-negotiable.

## Analysis

| Decision | Choice | Rationale | Rejected alternative |
|----------|--------|-----------|----------------------|
| **Emitter** | Pure-stdlib `encoding/json` over hand-written minimal SPDX-2.3 / CycloneDX-1.5 structs; slices pre-sorted, no maps (struct field declaration order = on-disk key order). `*json.Encoder` with `SetIndent("", "  ")`, `SetEscapeHTML(false)`, trailing newline. | Struct field order + pre-sorted slices give byte-stable output; zero random `serialNumber`/timestamps; no heavy transitive trees in the default build. | Use `spdx/tools-golang` / `cyclonedx-go` encoders — rejected: non-deterministic ordering, random `serialNumber`, wall-clock timestamps, and (per Phase 04 MVS) `go.mod` bloat even when tag-gated. |
| **Round-trip oracle (CI)** | The upstream libraries behind `//go:build omni_sbomvalidate` decode-then-revalidate omni's own output (proves schema-validity in-process), **plus** an external `syft convert omni.spdx.json -o cyclonedx-json` oracle run in CI. The validator build tag keeps those deps out of `omni` releases entirely. | Validates schema-correctness without contaminating the shipped binary or `go.mod`. Two independent oracles (in-process upstream decode + external `syft`) catch both schema drift and interoperability regressions. | No validation (trust hand-written structs blindly) — rejected: a typo in a JSON tag silently breaks downstream consumers; the gate guards against drift. |
| **purl correctness** | `pkg:golang/<lowercased-module-path>[@<canonical-version>]`. Version uses `semver.Canonical` when valid; pseudo-versions pass through unchanged; `+incompatible` is stripped before canonicalizing and re-appended; empty version → no `@` suffix. Replace directives resolve the purl to the **effective** (replacement) module; the original path/version are recorded for a comment/property. `std`@`go1.25.0` represents the toolchain. A purl round-trip test (build purl → re-parse path+version) is the drift guard. | Go module paths only contain `[a-z0-9.\-_~/]` once lowercased, so no percent-encoding is needed (asserted in tests). The "effective module" rule reflects "what actually shipped." | Percent-encode every segment / use Go's `module.EscapePath` `!`-escaping — rejected: purl uses RFC-3986 encoding, not Go's case-escaping, and Go paths never need it post-lowercasing; encoding the original (pre-replace) module — rejected: it misrepresents the shipped artifact. |
| **Determinism contract** | Identical input → identical output bytes. No UUID, no wall clock. Default creation timestamp is the literal epoch `1970-01-01T00:00:00Z`; `--source-date` (RFC-3339) overrides it. `documentNamespace` and the content hash derive from the first 16 hex chars of a `sha256` over the **sorted** list of purl strings, not randomness. CycloneDX `serialNumber` is omitted (the struct has no such field). | The whole point of the phase: golden-master pinnable, reproducible-build friendly, diff-stable SBOMs. The epoch default keeps output deterministic even with no flag. | Wall-clock `created` / random `serialNumber` / UUID namespace — rejected: breaks determinism, makes golden masters impossible, and defeats reproducible-build verification. |
| **Cross-package boundary** | `pkg/sbom/format.Document` is the **only** type other packages may import. Phase 6 `pkg/scan/` depends solely on `format.Document` and never on `pkg/sbom/model`, `pkg/sbom/collect`, or `pkg/sbom/purl`. `format` carries no `// Experimental:` marker (stable v1.0 boundary); `model`/`collect`/`purl` stay `// Experimental:` until Phase 3 triage. | A single stable boundary lets the internal model, collectors, and purl logic evolve freely without breaking downstream importers. | Expose `model.SBOM` directly to `pkg/scan/` — rejected: couples Phase 6 to an experimental, churn-prone internal representation. |

## Consequences

- **Default build stays lean and pure-Go.** `omni` releases never compile `spdx/tools-golang` or `cyclonedx-go`; the only new direct require is promoting `golang.org/x/mod` (already in `go.sum`) for `semver`, adding no transitive cost.
- **Determinism is enforceable.** Golden-master tests can pin exact bytes; the determinism contract (sorted slices, struct order, epoch default, sha256 content hash) makes a non-deterministic regression a test failure, not a silent drift.
- **Validation is opt-in and quarantined.** Schema-validity is proven in CI via the `omni_sbomvalidate` tag and external `syft convert`, but invoking `--validate` (or any validator path) **without** the build tag returns `cmderr.ErrUnsupported` (exit 6) rather than silently passing.
- **purl policy is locked.** The normalization rules (canonicalize, preserve `+incompatible`, pass pseudo-versions, drop `@` on empty, effective-module on replace) are fixed and guarded by a round-trip test; changing them is a deliberate, reviewable edit.
- **Phase 6 has a stable contract.** `pkg/sbom/format.Document` is the committed boundary; `pkg/scan/` can build against it now, and the internal model/collectors remain free to change behind it.
- **`--sign` reuses `pkg/sign`.** No re-implementation of signing; the Phase 04 primitive signs the emitted SBOM bytes.
- **Trade-off accepted:** hand-written minimal structs cover only the fields omni emits, not the full SPDX/CycloneDX schemas. This is intentional for v1.0 (deterministic, sufficient-for-consumption SBOMs); the round-trip oracle guards against the hand-written subset drifting out of schema-validity.
