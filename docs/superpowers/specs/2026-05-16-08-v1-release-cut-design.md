## Design / Approach / Components

Ship v1.0 as signed, reproducible binaries for six target platforms with SBOM and SLSA provenance generated by omni itself — dogfood the supply chain before announcing it.

**Expected components:**
- `pkg/` API freeze tagged at phase entry; 30-day CLAUDE.md deprecation protocol applies to every breaking change from that point forward.
- GoReleaser pipeline producing signed binaries for: `linux/amd64`, `linux/arm64`, `darwin/amd64`, `darwin/arm64`, `windows/amd64`, `windows/arm64`.
- Each binary signed with `omni sign` (dogfooding Phase 4 primitive).
- Reproducible builds: `-trimpath` + `-buildvcs=true`; CI dual-build comparison job fails on any reproducibility drift.
- Release assets: SBOM generated by `omni sbom` + SLSA v1.0 provenance attestation generated by `omni attest` at the ADR-pinned honest level.
- Release notes generated from conventional commits; include an explicit "What's NOT protected against" section linking `.planning/research/PITFALLS.md`.
- Announcement explicitly states target audience ("me + my CI/CD pipelines") — no overclaiming of general-purpose fitness.

## Rationale & Decisions

| Decision | Rationale |
|----------|-----------|
| Dogfood sign/sbom/attest in release | Validates supply chain primitives on the most important artifact |
| Reproducibility enforced in CI | Pitfall 6 — reproducibility drift caught before announce |
| Honest announcement | Pitfall 15 — no audience overclaim; explicit scope statement required |
| API freeze at phase entry | Establishes stable library contract; deprecation protocol prevents breaking consumers |
| Six platform targets | Linux, macOS, Windows × amd64 + arm64 — full cross-platform commitment |
